# 24 game001 demo bootstrap 任务报告

## 实现摘要

已新增 `apps/game001`，作为 Pixi slot demo app 接入 `@slotclientengine/logiccore`、`@slotclientengine/rendercore`、`@slotclientengine/netcore` 和当前 `assets/game001`、`assets/gamecfg/game2.json`、`assets/symbols` 资源。

核心行为：

- 页面层只创建 `.game001-page` 和 `.game001-frame`，`.game001-frame` 固定为背景图像素尺寸 `941px x 1672px`，canvas 占满 frame，外层只做水平居中，垂直从顶部开始显示，并按页面尺寸自动缩放。宽度允许时优先占满屏幕高度，宽度过窄时按宽度缩小以保证完整可见。
- Pixi stage 固定为 `941 x 1672`。
- 背景、logo、主转轮背景、副转轮背景按任务坐标放置。
- 主转轮按完整 `5 x 5` scene 解析、反查停轴和最终验收，视觉只通过 viewport mask 显示中间一行和上下相邻半行。
- 运行态读取并校验 `assets/symbols/symbol-composites.json` 与 `symbol-state-textures.manifest.json` layered normal 一致。
- `VITE_GAME001_SERVER_URL` 只接受 `ws://` / `wss://` live URL，不支持 `http(s)` replay。
- `game001` 缺省 live 配置已改为复用 `apps/gameclientcli` 的默认 `url`、`gamecode`、`token`、投注参数和请求超时；显式提供空字符串、非法 URL 或非正数仍会明确失败。
- live 初始画面只使用 server `defaultScene`；没有 live `defaultScene` 时不使用本地 fixture。
- Spin 点击通过 netcore live client 发送一次 spin，请求完成并严格解析 GMI 后才启动动画。
- Spin 返回后按本次 `totalwin/results` 判断是否需要最终 `collect()`，需要时显式调用一次 `client.collect()`，与 `apps/gameclientcli` 多次 spin 流程一致，避免中奖后下一次 spin 卡在 `SPINEND`。
- 重复点击、非法 env、资源缺失、netcore warning/error、意外断线、非法 GMI、非法 scene、停轴反查失败和最终 scene 不一致都会明确失败。

## netcore 修改说明

`packages/netcore` 增加 `SlotcraftClientOptions.autoCollectIntermediateResults?: boolean`，默认保持原行为 `true`。`game001` 当前显式保留 `true`，让 netcore 继续自动 collect 多段结果的中间段；game001 另外按当前 spin 的 `totalwin/results` 判断并发送最终 `collect()`。

同时 `packages/netcore/package.json` 的 `exports["." ]` 增加 `default` 条件，提升 Vite/browser bundler 消费包入口的兼容性。game001 源码仍通过 `@slotclientengine/netcore` 包名导入。

## 资源契约校验

`apps/game001/src/assets.ts` 运行时校验：

- `symbol-composites.json.version === 1`
- `SC`、`RS`、`X2`、`X5`、`X10` 必须存在
- composite layers 必须非空、文件名匹配 `{symbol}-{index}.png`、从 0 连续
- manifest 中同 symbol 的 `normal.kind` 必须为 `layered`
- manifest layered normal layers 必须与 composite layers 完全一致
- layer 文件不会成为顶层 symbol
- 未在 manifest 声明的状态贴图会失败
- 当前至少要求 `spinBlur`

单元测试覆盖了当前 manifest 下 `SC` normal layered、layer 排除、缺少 `spinBlur`、未知状态、composite 版本错误、layer 不连续、manifest/composite 不一致等路径。

## 服务端模式

运行态只接受 `ws://` 或 `wss://` live URL。`http://`、`https://`、相对 URL、显式空字符串都会明确失败；缺省 URL 会使用与 `apps/gameclientcli` 一致的默认 live 地址。

单元测试 mock client 覆盖：

- `connect -> enterGame -> defaultScene`
- 无 `defaultScene` 时不使用 fixture
- spin 参数为 `bet/lines/times`
- `createGameLogicFromGmi()` meta 使用请求 `bet/lines`、返回 `totalwin`、live `userInfo.gameid`
- `gameid` 缺失时不伪造
- 缺少 `gmi` / `totalwin` / `results` 失败
- `results` 与 `gmi.replyPlay.results.length` 不一致失败
- logger.warn、noticemsg2、意外 disconnect 失败
- 重复 spin 请求失败

live 浏览器验收：按用户本次要求“不需要你做浏览器验收”，未执行。因此 live 服务器 URL 实际联通、点击 Spin 真实发送 `gamectrl3/spin`、真实返回 GMI 后视觉旋转这些 manual/browser 项未验收，不能标记为完整 live 浏览器验收完成。
