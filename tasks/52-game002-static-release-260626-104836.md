## 3. 最终 URL query 合同

`game002` 静态发布版只从 `window.location.search` 读取运行配置。`main.ts` 已改为：

```ts
parseGame002FrameworkConfigFromQuery(window.location.search, {
  pageProtocol: window.location.protocol,
});
```

必需参数：

| 参数 | 规则 |
| --- | --- |
| `serverUrl` | 非空，合法 URL，只接受 `ws://` 或 `wss://`；HTTPS 页面禁止 `ws://` |
| `gamecode` | 非空字符串 |
| `token` | 非空字符串 |
| `businessid` | 非空字符串 |
| `clienttype` | 非空字符串 |
| `jurisdiction` | 非空字符串 |
| `language` | 非空字符串 |
| `bet` | 有限正数 |
| `lines` | 有限正数 |
| `times` | 有限正数 |
| `autonums` | 整数，允许 `-1` |
| `requestTimeoutMs` | 有限正数 |

失败规则已覆盖：

- 缺少必需参数会失败。
- 参数为空或 trim 后为空会失败。
- 已知参数重复出现会失败。
- 非 WebSocket `serverUrl`、非法 URL、HTTPS 页面使用 `ws://` 会失败。
- 非法数字和非整数 `autonums` 会失败。
- token 未正确 URL encode 导致空白解析时会失败；错误消息不输出 token 实值。
