## 完成内容

### rendercore

- 为 `scene-layout` 增加严格的有向 mode transition schema：每条边显式绑定一个 official Spine resource、animation、唯一 event 以及各 active variant 的 art-space placement。
- manifest/parser 对未知字段、自环、重复边、缺失 mode/resource/variant、非 official Spine、错误 animation/event、重复 event occurrence 和闭包外资源显式失败。
- Spine runtime 每帧返回冻结且有序的 event snapshot；覆盖 `time=0`、大 delta、reset/play/destroy 清理和当前 entry 隔离。
- production runtime 固定拥有最上层 `scene-transition-overlay`。模式切换先准备目标背景和 symbol package，再播放 overlay；只在约定 event 边界原子提交背景、转轮数据、symbol package 和 displayed mode，动画完成后恢复 stable。
- 删除 game-mode 对 scene node state machine 的切换依赖和静默 fallback；低层 node state API 仅保留给独立使用者。
- 增加并发请求拒绝、缺边拒绝、目标 prepare、event 前后快照、超大 delta、viewport/orientation placement、overlay 层级、destroy 等回归测试。

### gamelayouteditor

- 游戏模式背景改为稳定 loop-only 绑定；移除旧的 mode scene-node state 编辑入口和命令。
- 增加 `转场` workspace，并形成六个固定页签：`资源 | 布局 | 转场 | Symbols | BigWin | 项目`。
- 转场资源、动画和事件均使用严格下拉选择；事件元数据从 Spine skeleton 读取 occurrence time/count，重复事件不允许选择并给出诊断。
- 支持有向边创建/删除、resource/animation/event 编辑、各 variant overlay placement 编辑、production preview 播放和详细 runtime snapshot。
- 补齐 mode/resource rename/delete 的 transition 引用维护与阻断；同一 logical Spine 在不同 mode 中建立独立 scene node。
- ZIP export/import 支持 transition-only resource、owned payload content-addressed materialization、相同签名去重和确定性 round-trip；旧 mode state-machine package 显式拒绝且不伪迁移。
- preview 加入 stale request token，rebuild/import/destroy 后旧异步请求不能回写新会话。

### 文档

- 更新 `apps/gamelayouteditor/README.md`、`docs/scene-layout-manifest.md` 和 `agents.md`，明确稳定背景、顶层转场 overlay、事件边界原子切换以及旧 state-machine fallback 已移除。
