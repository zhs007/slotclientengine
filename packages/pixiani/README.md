# [projname]

这是一个干净的 PixiJS v8 + GSAP 动画项目模板，用于构建可复用、可测试、可持续扩展的程序化动画工程。

这个模板刻意不包含任何示例动画代码、示例资源或示例测试。它提供的是可直接继续开发的工程骨架，而不是演示项目。

## 模板目标

- 用统一生命周期管理动画实体
- 用对象池复用动画实例
- 用实体管理器统一更新和回收
- 用 Vitest 为动画实体补齐测试
- 用干净的工程结构支持持续迭代

## 目录结构

- `src/core`
  - 核心实体系统与对象池逻辑
  - 这是模板的基础设施层
- `src/ani`
  - 动画实体实现目录
  - 模板初始化时为空，等待按任务新增
- `src/main.ts`
  - 最小运行入口
  - 已初始化 Pixi 应用、图层和更新循环，但不包含任何示例动画
- `tests/core`
  - 核心基础设施测试
- `tests/ani`
  - 动画实体测试目录
  - 模板初始化时为空，等待和 `src/ani` 一起补齐

## 开发方式

- 先理解 `src/core` 提供的生命周期、池化与更新机制
- 在 `src/ani` 中新增动画实体
- 在 `tests/ani` 中为对应实体补齐测试
- 在 `src/main.ts` 中按实际需求接入动画、资源和图层

## 重要约束

- 模板不提供示例动画，请不要假设已有现成动画可以改
- 新动画应遵循统一生命周期，并与对象池、实体管理器配合
- 新资源应按真实任务需求引入，不保留演示用素材
- 交付前应通过 `typecheck`、`lint`、`test`、`build`

## 相关说明
